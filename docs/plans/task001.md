# Задание на реализацию редактирования настроек JRFBoardData

## Описание задачи

Необходимо реализовать редактирование настроек JRFBoardData в экране настроек приложения.

## Требования к реализации

В экране настроек `src/pages/Settings.tsx` необходимо добавить 4 блока для редактирования данных:

1. **Делитель для Reach**
    - Поле целое положительное число от 1 до 1_000_000
    - Имя поля: `reachDivider`

2. **Блок категорий Impact**
    - Для каждой категории задаётся имя и описание категории
    - В каждой категории есть уровни по `JRFBoardDataImpactCategoryLevelKeys`
    - Все уровни для каждой категории должны быть заполнены
    - Для каждого уровня задаются:
        1. Имя уровня (`names`)
        2. Описание уровня (`descriptions`)
        3. Значение для уровня (`values`)

3. **Массив confidences**
    - Каждый элемент содержит имя и значение
    - Число элементов от 1 до 10

4. **Описание для поля effort**
    - Поле `effortDescription`

## Правила валидации

1. `reachDivider` - задан, положительное число от 1 до 1_000_000
2. `impactCategories[]` - от 1 до 10 элементов
    - `impactCategories[].name` - не пустая строка
    - `impactCategories[].description` - не пустая строка
    - `impactCategories[].names[]` - не пустые строки для всех значений `JRFBoardDataImpactCategoryLevelKeys`
    - `impactCategories[].descriptions[]` - не пустые строки для всех значений `JRFBoardDataImpactCategoryLevelKeys`
    - `impactCategories[].values[]` - значения от 0 до 100 для всех значений `JRFBoardDataImpactCategoryLevelKeys`
3. `confidences[]` - от 1 до 10 элементов
    - `confidences[].name` - не пустая строка
    - `confidences[].value` - значение от 0 до 100
4. `effortDescription` - не пустая строка

## Требования к реализации

- При валидации данных, если они невалидны, кнопка закрытия с сохранением должна быть недоступна
- Поле с ошибкой должно подкрашиваться красным
- Добавить кнопку закрытия без сохранения, которая доступна всегда
- Данные должны читаться при открытии формы
- Если нажата кнопка закрыть с сохранением, данные должны быть сохранены на доску
- Методы чтения и сохранения вынести в утилиты `src/utils/JiraUtils.ts`

## Структура данных

Данные хранятся в типе `JRFBoardData` из файла `src/types/JiraRiceFarmTypes.ts`.

## API для работы с данными

- Получение списка ключей: `GET /rest/agile/1.0/board/{boardId}/properties`
- Получение значения ключа: `GET /rest/agile/1.0/board/{boardId}/properties/{propertyKey}`
- Запись значения: `PUT /rest/agile/1.0/board/{boardId}/properties/{propertyKey}`

## Особенности реализации

- Использовать существующие типы данных из `src/types/JiraRiceFarmTypes.ts`
- Все валидации должны выполняться на стороне клиента
- При открытии формы данные должны быть загружены из Jira
- При сохранении данные должны быть записаны в Jira
