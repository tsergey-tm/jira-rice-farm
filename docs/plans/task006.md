# Задача: Сортировка задач бэклога по значению RICE

## Цель

Реализовать возможность сортировки списка задач в бэклоге Jira по их весу (значение RICE) при нажатии на новую иконку в
заголовке группы бэклога.

## Технические детали реализации

### 1. Инъекция интерфейса

Необходимо доработать модификатор `src/modifiers/issuesList.ts`.
Необходимо добавить кнопку-иконку для запуска процесса сортировки в заголовок контейнера бэклога.

- **Селектор для вставки**:
  `#ghx-content-group > div.ghx-backlog-group > div.ghx-backlog-container > div.ghx-backlog-header`
- **Тип элемента**: Кликабельный элемент (например, `button` или `span` с иконкой).

### 2. Алгоритм сортировки (обработчик клика)

При нажатии на иконку необходимо выполнить следующие шаги:

1. **Сбор данных о задачах**:
    * Найти все элементы задач по селектору:
      `#ghx-content-group > div.ghx-backlog-group > div.ghx-backlog-container > div.ghx-issues > div.js-issue`.
    * Для каждого элемента извлечь `issueKey` из атрибута `data-issue-key`.
    * Получить `boardId` с помощью функции `getBoardId()` (из `src/utils/JiraUtils.ts`).
    * Загрузить данные доски (`boardData`) с помощью функции `jiraBoardDataStore()` (из `src/data/JiraData.ts`).

2. **Загрузка данных задач**:
    * Для каждого найденного `issueKey` вызвать функцию `getIssueData(issueKey, boardId)` (из `src/utils/JiraUtils.ts`).
    * **Важно**: Используйте `Promise.all`, чтобы дождаться загрузки данных для всех задач перед началом сортировки.

3. **Расчет и сортировка**:
    * Используя полученные `boardData` и `issueData` для каждой задачи, рассчитайте `riceValue` через функцию
      `calcRICEValues(boardData, issueData)` (из `src/utils/RICEUtils.ts`).
    * Отсортируйте DOM-элементы задач (`div.js-issue`) по убыванию значения `riceValue`.
    * **Обработка исключений**: Если `riceValue` отсутствует, является `NaN` или не определено, такие задачи должны быть
      перемещены в конец списка.

4. **Обновление DOM**:
    * Переставьте отсортированные элементы задач внутри их родительского контейнера:
      `#ghx-content-group > div.ghx-backlog-group > div.ghx-backlog-container > div.ghx-issues`.
    * **Критическое требование**: При сортировке **нельзя** менять порядок элементов, которые не являются
      `div.js-issue` (например, разделители, пустые строки или другие служебные элементы внутри контейнера
      `div.ghx-issues`). Они должны сохранять свои позиции относительно друг друга и относительно "слотов" для задач.

## Используемые функции

- `getBoardId()` — `src/utils/JiraUtils.ts`
- `jiraBoardDataStore()` — `src/data/JiraData.ts`
- `getIssueData(issueKey, boardId)` — `src/utils/JiraUtils.ts`
- `calcRICEValues(boardData, issueData)` — `src/utils/RICEUtils.ts`

## Критерии приемки

1. Иконка отображается в заголовке бэклога.
2. При клике список задач перестраивается согласно убыванию RICE.
3. Задачи без веса уходят в конец списка.
4. Служебные элементы внутри контейнера задач не перемещаются.
5. Код соответствует существующим паттернам проекта.
