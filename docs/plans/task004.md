# Задача: Отображение значения RICE в BoardIssueCard

## Цель

Реализовать отображение рассчитанного показателя RICE на каждой карточке задачи в представлении доски (board view).

## Детали реализации

### 1. Компонент: `src/pages/Issue/BoardIssueCard.tsx`

В настоящее время компонент принимает `issueKey` в качестве пропса. Вам необходимо реализовать логику получения и
отображения значения RICE.

#### Логика:

1. **Получение данных задачи**: Используйте `getIssueData(issueKey)` из `src/utils/JiraUtils.ts` внутри хука
   `useEffect`, чтобы получить свойства задачи.
2. **Доступ к данным доски**: Получите данные доски из `jiraBoardDataStore.jrfBoardData`, расположенного в
   `src/data/JiraData.ts`.
3. **Расчет RICE**: Как только оба набора данных (`issueData` и `boardData`) будут доступны, используйте
   `calcRICEValues(boardData, issueData)` из `src/utils/RICEUtils.ts`, чтобы получить `riceValue`.

#### Правила рендеринга:

* **Отсутствие данных**: Если данные доски (`jrfBoardData`) или полученные данные задачи равны `null` или `undefined`,
  компонент должен вернуть пустой `div` (`<div />` или `<div></div>`).
* **Отображение RICE**: Если оба набора данных доступны и `riceValue` является допустимым числом (не `NaN`):
    * Отобразите `div`, содержащий текст: `RICE = <value>`.
    * Этот `div` должен быть расположен в **верхнем правом углу** своего родительского контейнера.
    * Имя класса div = jira-rice-farm-board-issue-card-rice
    * Используйте `position: relative`, `top: 0.5em` и `right: 0.5em` для класса этого `div`.

### 2. Импортируемые зависимости

- `useState`, `useEffect` из `'react'`
- `getIssueData` из `'@/utils/JiraUtils'`
- `jiraBoardDataStore` из `'@/data/JiraBoardData'`
- `calcRICEValues` из `'@/utils/RICEUtils'`

## Шаги верификации

1. Проверьте, что значение RICE отображается в верхнем правом углу карточки задачи при наличии данных.
2. Проверьте, что ничего (пустой div) не отображается, если у задачи нет свойств RICE или отсутствуют данные доски.
3. Убедитесь в отсутствии ошибок в консоли, связанных с обращением к `null`/`undefined`.
4. Запустите `npm run lint` и `npm run typecheck` для проверки качества кода.
