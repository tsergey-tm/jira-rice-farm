# Требования к экрану редактирования задачи (Issue/BoardIssueEditor.tsx)

## Общее описание

Необходимо создать экран редактирования данных задачи по технологии RICE для Jira. Экран должен быть реализован в файле
`src/pages/Issue/BoardIssueEditor.tsx`.
Ты должен редактировать только файлы `src/pages/Issue/BoardIssueEditor.tsx` и `src/pages/Issue/BoardIssueEditor.css`.

## Структура данных

Экран будет работать с данными из следующих типов:

- `JRFIssueData` (из `src/types/JiraRiceFarmTypes.ts`) - данные задачи для сохранения
- `JRFBoardData` (из `src/types/JiraRiceFarmTypes.ts`) - данные доски для формирования полей

## Поля формы

### 1. Reach (Охват)

- Тип: селектор выбора между "Размер выборки/Средний доход" и "Предполагаемое увеличение прибыли"
- При выборе "Размер выборки/Средний доход":
    - Поле "Размер выборки" (целое число от 0 до 100,000,000)
    - Поле "Средний доход выборки" (целое число от 0 до 1,000,000,000)
- При выборе "Предполагаемое увеличение прибыли":
    - Поле "Предполагаемое увеличение прибыли" (целое число от 0 до 1,000,000,000)

### 2. Impacts (Влияние)

- Для каждой категории из `JRFBoardData.impactCategories` создается селектор
- Селектор содержит значения из `JRFBoardDataImpactCategory.names`
- Рядом с каждым селектором отображается описание из `JRFBoardDataImpactCategory.descriptions`

### 3. Confidence (Уверенность)

- Селектор выбора значения из `JRFBoardData.confidences.name`

### 4. Effort (Сложность)

- Поле ввода числа от 1 до 100

## Валидация

1. **Reach**: выбран один из типов
    - Если "Размер выборки/Средний доход":
        - Размер выборки: от 0 до 100,000,000
        - Средний доход выборки: от 0 до 1,000,000,000
    - Если "Предполагаемое увеличение прибыли":
        - Предполагаемое увеличение прибыли: от 0 до 1,000,000,000
2. **Impacts**: заполнены все значения из `JRFBoardDataImpactCategory.name`
3. **Confidence**: значение выбрано, не пустое
4. **Effort**: число от 1 до 100

## Поведение формы

- Использовать react-modal для отображения формы
- Закрытие формы по ESC и клику на серую область разрешено
- Кнопка сохранения недоступна, если форма не валидна
- Кнопка закрытия без сохранения всегда доступна

## Интеграция с системой

- Использовать методы `getIssueData` и `setIssueData` из `src/utils/JiraUtils.ts`
- Получать данные доски через `jiraBoardDataStore` из `src/data/JiraData.ts`

## Дизайн

Следовать стилям из `src/pages/Settings/Settings.tsx` и `src/pages/Settings/Settings.css`:

- Использовать те же цвета, шрифты и стили
- Структура формы должна быть похожа на Settings
- Кнопки должны иметь аналогичный дизайн

## Функциональность

- Загрузка данных задачи при открытии формы
- Сохранение данных задачи при нажатии кнопки сохранения
- Не нужны алерты при успешных действиях
- Обработка ошибок загрузки/сохранения
